# PRD: Klip 全量实现 Clipy 核心能力（Tauri 跨平台）

> 创建日期: 2026-03-03
> 最近更新: 2026-03-03

## Overview
Klip 目标是实现与 Clipy 等价的核心体验：`History + Snippets + Menu Bar/Tray + Global Hotkey + Search + Direct Paste`。
产品采用 Tauri 架构，支持 macOS 与 Windows。当前版本仅支持纯文本剪贴板，不做隐私过滤策略与云同步。

## Goals
- 在 macOS/Windows 上提供与 Clipy 一致的高频工作流。
- 用户可通过全局快捷键在任意应用中快速检索并粘贴历史或片段。
- 历史记录与片段可稳定持久化，本地离线可用。
- UI 与交互达到可日常替代 Clipy 的水平。

## Quality Gates
These commands must pass for every user story:
- `npm run lint`
- `npm run typecheck`
- `npm run test`
- `npm run build`
- `npm run test:e2e`
- `cargo check --manifest-path src-tauri/Cargo.toml --locked`

For UI stories, also include:
- 在 macOS 与 Windows 各执行一次人工界面验证（打开面板、搜索、选择、粘贴、关闭流程）。

## Rollout Plan

### Phase 1: MVP
目标：先形成可用闭环（文本历史 + 快速粘贴）。

范围：
- US-001 剪贴板文本监听
- US-002 历史持久化与容量控制
- US-003 托盘/Menu Bar 常驻入口
- US-004 全局快捷键呼出
- US-005 历史搜索与键盘导航
- US-006 选择即粘贴（Direct Paste）

### Phase 2: Clipy 等价
目标：达到可替代 Clipy 的核心能力。

范围：
- US-007 Snippets + 文件夹管理
- US-008 Snippets 快速粘贴
- US-009 History/Snippets 双模式入口
- US-010 设置中心
- US-011 跨平台打包与安装验证
- US-012 自动化测试与回归基线

### Phase 3: 增强
目标：在等价基础上做差异化增强（单独立项）。

候选方向：
- 隐私过滤（应用排除、关键词/正则、临时暂停）
- 更多内容类型（图片/文件/富文本）
- 同步与高级模板能力

## User Stories

### US-001: 剪贴板文本监听
**Description:** As a user, I want Klip to capture copied text automatically so that I can reuse clipboard history later.

**Acceptance Criteria:**
- [ ] 在 macOS 和 Windows 监听系统剪贴板变更。
- [ ] 仅采集纯文本内容，非文本内容不入库。
- [ ] 避免应用自身写回导致的循环采集。
- [ ] 监听服务在应用启动后 2 秒内进入可用状态。

### US-002: 历史记录持久化与容量控制
**Description:** As a user, I want history to persist across restarts so that copied text is not lost.

**Acceptance Criteria:**
- [ ] 历史数据持久化到本地存储，重启后可恢复。
- [ ] 支持可配置最大历史条数，超过上限时按 FIFO 淘汰。
- [ ] 历史项包含文本内容、创建时间、来源元信息（若可获取）。
- [ ] 首次安装存在默认容量值并可在设置中修改。

### US-003: 托盘/Menu Bar 常驻入口
**Description:** As a user, I want a tray/menu bar entry so that I can access Klip quickly.

**Acceptance Criteria:**
- [ ] macOS 使用 menu bar 图标，Windows 使用 system tray 图标。
- [ ] 点击图标可打开历史/片段面板。
- [ ] 提供“退出应用”入口并可正常释放后台资源。

### US-004: 全局快捷键呼出面板
**Description:** As a user, I want global shortcuts so that I can open Klip without switching context.

**Acceptance Criteria:**
- [ ] 支持配置历史面板全局快捷键。
- [ ] 快捷键冲突时给出明确错误提示并允许重新设置。
- [ ] 快捷键触发后面板在当前工作流可见位置弹出。
- [ ] 按 `Esc` 可关闭面板并回到原前台应用。

### US-005: 历史搜索与键盘导航
**Description:** As a user, I want fast search in history so that I can find old copied text quickly.

**Acceptance Criteria:**
- [ ] 输入关键字后实时过滤结果，结果刷新延迟小于 100ms（1k 条以内）。
- [ ] 支持上下方向键选择结果。
- [ ] 支持 `Enter` 触发选中项动作。
- [ ] 空查询时展示最近历史列表。

### US-006: 选择即粘贴（Direct Paste）
**Description:** As a user, I want selected items to paste directly into the active app so that I save one extra step.

**Acceptance Criteria:**
- [ ] 在历史面板选择条目后，文本直接粘贴到当前前台应用。
- [ ] 粘贴成功后面板自动关闭。
- [ ] 粘贴失败时有可见错误反馈，并至少回退到复制到系统剪贴板。
- [ ] macOS 与 Windows 粘贴行为一致可预测。

### US-007: Snippets 文本片段与文件夹管理
**Description:** As a user, I want to manage reusable text snippets in folders so that repetitive input becomes faster.

**Acceptance Criteria:**
- [ ] 支持创建、编辑、删除文本片段。
- [ ] 支持创建、重命名、删除文件夹并将片段归档到文件夹。
- [ ] 片段与文件夹重启后可恢复。
- [ ] 删除操作有确认机制，避免误删。

### US-008: Snippets 快速粘贴
**Description:** As a user, I want to quickly paste snippets so that templates can be inserted in one action.

**Acceptance Criteria:**
- [ ] 支持通过片段面板搜索并选择片段。
- [ ] 选择片段后执行 direct paste 并关闭面板。
- [ ] 支持为片段配置快捷触发方式（全局快捷键或片段别名触发）。
- [ ] 触发冲突时有明确提示与修复路径。

### US-009: 双模式入口（History/Snippets）
**Description:** As a user, I want both history and snippets in one product so that workflows stay unified.

**Acceptance Criteria:**
- [ ] 面板内可在 History 与 Snippets 间快速切换。
- [ ] 支持键盘切换模式，不依赖鼠标。
- [ ] 记住上次模式并在下次呼出时恢复。

### US-010: 设置中心
**Description:** As a user, I want configurable behavior so that Klip can match my habits.

**Acceptance Criteria:**
- [ ] 可配置历史容量上限。
- [ ] 可配置 History/Snippets 快捷键。
- [ ] 可配置开机自启动开关。
- [ ] 设置修改后即时生效并持久化。

### US-011: 跨平台打包与运行验证
**Description:** As a user, I want installable builds on both target platforms so that Klip is actually usable.

**Acceptance Criteria:**
- [ ] 提供 macOS 与 Windows 可安装产物。
- [ ] 安装后首次启动可正常拉起监听与托盘。
- [ ] 卸载流程不会残留影响运行的后台进程。
- [ ] 发布说明包含已知权限需求与限制。

### US-012: 自动化测试与回归基线
**Description:** As a maintainer, I want automated regression checks so that parity features remain stable.

**Acceptance Criteria:**
- [ ] 为核心状态管理和数据层补齐 Vitest 单元测试。
- [ ] 覆盖至少 4 条 e2e 主流程：历史采集、历史搜索、direct paste、snippet direct paste。
- [ ] CI 在 macOS 与 Windows 上执行关键自动化检查。
- [ ] 变更代码覆盖率目标为 >=80%，不足时附人工验证清单。

## Functional Requirements
1. FR-1: 系统必须在 macOS 与 Windows 上监听剪贴板文本变更。
2. FR-2: 系统必须仅保存纯文本内容，其他类型本期忽略。
3. FR-3: 系统必须将历史记录本地持久化，并支持容量上限。
4. FR-4: 系统必须提供 tray/menu bar 常驻入口。
5. FR-5: 系统必须提供可配置全局快捷键呼出面板。
6. FR-6: 系统必须支持历史搜索、键盘导航和回车选择。
7. FR-7: 系统必须在用户选择历史项后执行 direct paste 并自动关闭面板。
8. FR-8: 系统必须支持 snippets 的 CRUD 与文件夹组织。
9. FR-9: 系统必须支持 snippet 搜索与 direct paste。
10. FR-10: 系统必须提供 History/Snippets 双模式切换。
11. FR-11: 系统必须提供设置中心，支持快捷键、容量、自启动配置。
12. FR-12: 系统必须支持 macOS 与 Windows 打包发布。
13. FR-13: 系统必须具备可重复执行的单元测试与 e2e 回归测试。
14. FR-14: 系统必须在粘贴失败时提供可见反馈与降级策略。
15. FR-15: 系统必须保证常用查询场景的交互响应速度（1k 条以内搜索 <100ms）。

## Non-Goals (Out of Scope)
- 图片、文件、富文本剪贴板支持。
- 隐私过滤策略（应用黑名单、关键词/正则过滤、临时暂停监听）。
- 多设备云同步（iCloud/自建同步）。
- Linux 平台支持。
- 插件系统与脚本扩展能力。
- OCR、翻译、AI 补全等增强功能。

## Technical Considerations
- 前端建议在 `src/` 中按 `features/history`, `features/snippets`, `features/settings` 组织。
- 后端能力建议在 `src-tauri/` 中抽象剪贴板监听、粘贴执行、全局快捷键模块。
- 需要处理平台差异：macOS 辅助功能权限、Windows 输入模拟权限与前台窗口聚焦行为。
- 建议为 direct paste 建立统一接口，屏蔽平台注入方式差异。
- e2e 建议引入可跨平台运行的自动化方案，并为桌面场景提供稳定选择器策略。

## Success Metrics
- 核心能力对齐清单（History/Snippets/Menu/Hotkey/Search/Paste）完成率 100%。
- `Enter` 到完成粘贴的 P95 延迟 <= 300ms。
- 1k 条历史数据下搜索交互 P95 <= 100ms。
- 发布后首月核心流程崩溃率 < 0.5%。
- 内测用户中“可替代 Clipy 日常使用”反馈 >= 80%。

## Open Questions
- OQ-1: 片段快捷触发的最终交互未完全定稿（全局快捷键/别名命令/两者并存）。
- OQ-2: Windows 下 direct paste 的最优注入方案需在 PoC 后定版（稳定性优先）。
